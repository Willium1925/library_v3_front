現代瀏覽器為此提供了一個標準的解決方案：KeyboardEvent.isComposing 屬性。
•
當使用者正在使用 IME (輸入法) 進行組字時（例如，打注音或拼音，還在選字階段），isComposing 的值會是 true。
•
當使用者按下 Enter 鍵確認選字後，isComposing 會變回 false。
•
如果使用者不是在使用 IME，isComposing 恆為 false。
因此，我們的策略是修改事件處理函式，只有在 isComposing 為 false 時，才執行 Enter 鍵的送出邏輯。

原本
````
<td>
    <input 
        v-if="editing === item.id" 
        v-model="item[nameField]"
        @keyup.enter="save(item)"
        />
    <span v-else>{{ item[nameField] }}</span>
</td>

const save = async (item) => {
    try {
        const data = { [props.nameField]: item[props.nameField] }
        await props.api.update(item.id, data)
        alert('更新成功')
        editing.value = null
        loadAll()
    } catch (error) {
        alert('更新失敗：' + error)
    }
}
````

修正
````
<td>
    <input
        v-if="editing === item.id"
        v-model="item[nameField]"
        @keydown.enter.prevent="save(item, $event)"
    />
    <span v-else>{{ item[nameField] }}</span>
</td>

const save = async (item, event) => {
    if (event && event.isComposing) {
        return;
    }
    try {
        const data = { [props.nameField]: item[props.nameField] }
        await props.api.update(item.id, data)
        alert('更新成功')
        editing.value = null
        loadAll()
    } catch (error) {
        alert('更新失敗：' + error)
    }
}
````
